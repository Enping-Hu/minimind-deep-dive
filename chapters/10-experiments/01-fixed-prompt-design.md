# 固定 prompt 实验设计

前面九章读的是源码和机制。这一章用一组真实实验，看不同训练阶段的权重在相同输入下到底表现出什么差异。本节先讲实验怎么设计——它不需要改任何训练代码，用项目自带的 `eval_llm.py` 就能跑，任何人拿到权重都能复现。

## 实验要回答的问题

一句话：**同一批 prompt、同一组解码参数下，不同训练阶段的权重在指令遵循、回答完整性、格式稳定性、生成长度上有什么差异？**

具体拆成几个观察点：

- `pretrain` 是否更像续写文本，而不是对话回答？
- `full_sft` 是否明显更会按用户问题回答？
- `dpo` / `ppo_actor` / `grpo` 是否表现出对齐后的风格或格式变化？
- 不同权重是否有长度、重复、格式异常等问题？

这个实验**不证明某个方法一定更强**，只做小规模行为观察——把前面学的训练信号主线（[第 7 章](../07-ppo-grpo/05-training-signal-and-unified-source.md)）落到可见的模型行为上。

## 为什么从这个实验起步

不建议一上来跑 PPO/GRPO 训练再对比：RL 依赖 reward model、显存和时间成本高、reward/KL/advantage 日志分析复杂。固定 prompt 输出对比更适合第一步——用现成 [eval_llm.py](../04-inference/02-eval-and-service.md)、不改训练脚本、直接观察行为、结果易整理。

## 入口与内置 prompt

`eval_llm.py` 的 `--weight` 接受 `pretrain / full_sft / dpo / ppo_actor / grpo` 等前缀，运行选 `[0] 自动测试` 就依次跑内置的 8 条 prompt：

```python
prompts = ['你有什么特长？', '为什么天空是蓝色的', '请用Python写一个计算斐波那契数列的函数',
           '解释一下"光合作用"的基本过程', '如果明天下雨，我应该如何出门',
           '比较一下猫和狗作为宠物的优缺点', '解释什么是机器学习', '推荐一些中国的美食']
```

这 8 条覆盖了自我认知、事实问答、代码生成、过程解释、推理、对比、概念解释、推荐——足够做第一版行为对比。

## 控制变量

对比要公平，关键是**所有权重用完全相同的解码参数**：

| 变量 | 建议值 | 原因 |
|---|---|---|
| prompt 集合 | 同一批内置 8 条 | 输入一致 |
| `max_new_tokens` | 512 | 避免默认 8192 太长、也避免太短截断 |
| `temperature` | 0.85 | 控制随机性 |
| `top_p` | 0.95 | 与默认接近 |
| `historys` | 0 | 避免历史对话干扰 |
| seed | 2026 | `eval_llm.py` 内置 `setup_seed(2026)` |

唯一变量是 `--weight`（哪个阶段的权重）。

## 结论的边界

这类实验先天有几个限制，写结论时必须挂在前面：

1. **单次随机采样**：`do_sample=True`，即使固定 seed，换 prompt 或换权重路径仍是单条采样，不是统计意义上的确定结论。
2. **行为观察 ≠ 能力证明**：能看出「形式/风格变了」，但「事实更对」需要更系统的评测。
3. **训练曲线 ≠ 能力**：reward 在训练中上升，不等于模型能力提升（[03 节](03-eval-conclusions-sft-vs-rl.md) 会具体看到这一点）。

## 练习

1. 这个固定 prompt 实验要回答什么问题？它能证明「某方法更强」吗？
2. 对比不同阶段权重时，唯一该变的变量是什么？为什么其余都要固定？
3. 为什么不建议一上来就跑 RL 训练再对比，而是先做固定 prompt 输出对比？

<details>
<summary>参考答案</summary>

1. 同一批 prompt、同一组解码参数下，不同阶段权重在指令遵循/完整性/格式/长度上的差异；不能证明某方法更强，只做行为观察。
2. 唯一变 `--weight`（训练阶段）；其余（prompt、max_new_tokens、temperature、top_p、seed 等）固定才能保证差异来自权重而非解码设置。
3. RL 依赖 reward model、成本高、日志分析复杂；固定 prompt 对比用现成 eval_llm.py、不改训练脚本、直接观察行为、结果易整理，更适合第一步。
</details>
