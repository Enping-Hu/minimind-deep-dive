# 延伸：位置编码的演进——为什么最后是 RoPE

主线 [03-rope](../02-model/03-rope.md) 讲了 MiniMind 用的 RoPE 怎么旋转 Q/K，附录 [03-rope 长度外推](03-rope-length-extrapolation.md) 讲了 RoPE 之后怎么把上下文撑长。这一篇补最前面那一段：**RoPE 之前**，位置编码走过哪些路，为什么绝对位置不够用、相对位置怎么接棒，最后 RoPE 和 ALiBi 又各自收束到哪。读完这三篇，RoPE 的来龙去脉就闭环了。

本篇是**延伸 survey**，不绑定某段源码，但终点落回 MiniMind 的选择（v2/v3 都用 RoPE）。它能回答一个读源码时容易跳过、面试却常问的问题：**位置编码这么多种，为什么现在的开源 LLM 几乎都选了 RoPE？**

一句话主线：**位置编码的演进，是位置信息从「加在 embedding 上的坐标标签」一步步变成「写进 attention 打分里的相对关系」。**

## 为什么需要位置编码

self-attention 本身不区分先后。把整段序列一起送进去，`QK^T` 只看内容匹配，不知道谁在前谁在后——严格说它近似是 permutation-equivariant 的：把输入 token 整体打乱，模型看到的「谁和谁相似」会跟着一起乱，却意识不到原本是一句有语序的话。

但语言不是词袋，顺序本身承载语义：「他欠我一百万」和「我欠他一百万」token 几乎没变，意思却反过来。位置编码要做的，就是把「集合里的元素」重新变回「序列里的元素」。

位置关系其实有三类，别混成一件事（把四个 token 记成 `a1 a2 a3 a4`）：

- **绝对位置**：`a1` 是第一个，像坐标，「你在第几格」。
- **相对位置**：`a4` 在 `a2` 后面两位，像关系线索，「另一个词在我左边还是右边」。
- **距离**：`a1` 和 `a4` 相差三格，像依赖强度提示，「离我近还是远」。

一个好的位置方案，理想上要同时兼顾这三类。整条演进史，就是在这三者和一个额外目标——**长度外推**（能不能处理训练时没见过的更长句子）——之间反复权衡。

## 绝对位置编码：给每个位置贴坐标

绝对位置编码回答最直接的问题：当前 token 在整条序列里排第几。两条路线。

**正弦位置编码**（原始 Transformer）：直接把一组 sin/cos 值加到 embedding 上，位置在最早阶段就和内容融合。用不同频率的正余弦：高频维度区分近距离、低频维度覆盖长范围。好处是数值有界、由显式函数生成，理论上能算出训练没见过的更长位置。短板也明显：点积关系偏无向，只和位置差大小相关，不天然区分「前」和「后」；而且位置向量和 embedding 混在一起、再过后续投影矩阵后，那点相对结构还可能被冲淡。

**可学习绝对位置编码**（BERT）：直接开一个 `最大长度 × 隐藏维` 的参数矩阵，第 `i` 行是第 `i` 个位置的向量，随模型一起训。好处是能针对训练数据学出最合适的位置表示，对句长边界稳定的任务（分类、句对匹配）很好用。坏处是预训练只到某个长度（如 512），就只真正「认识」这些位置，再长要么没参数、要么只能插值，**长度外推通常很差**。

两条路线的取舍贯穿全程：正弦式先验强、理论可外推；可学习式更贴训练任务，但更依赖训练区间。早期 encoder 模型句长固定、只求训练区间内表现，爱用可学习绝对位置；当模型转向长上下文生成，长度外推从可选项变成硬需求，固定函数式又重新有吸引力。

## 相对位置编码：把位置写进 attention

绝对位置的根本局限是：很多语言规律更依赖「你在我前面还是后面、离我多远」，而不是「你在全句里恰好排第几」。主谓搭配、修饰关系、局部依存，常是一种相对关系。「虽然……但是……」，真正起作用的不是「但是」恰好在第 37 个 token，而是它离前面的「虽然」有多远。于是这条路线不再把位置加到 embedding，而是把**相对距离直接写进 attention 计算**。

几个代表，能看清这条线怎么一步步简化：

- **RPR（Relative Position Representation，Google）**：给「相差 +1」「相差 −2」各配一个位置向量，模型关注的从「这词在第 57 位」变成「它离当前查询前两格还是后三格」。通常做截断，因为语言里常用的多是有限范围的相对关系。它的核心优点是**平移不变性**：「not good」和「not very good」无论出现在句首句中，「否定词离形容词很近」这层关系是共通的。
- **Transformer-XL / XLNet**：把 attention 公式完全展开，把绝对位置项替换成相对位置向量（沿用正弦式生成、更易处理长上下文），并用可训练向量吸收部分偏置。它做了一个影响深远的判断：**相对位置信息主要加到 attention 分数这一侧，不再去改 value 输出**——位置的作用是帮模型决定「看谁、看多重」，不必混进最终输出。相对位置编码从此越来越像一种「注意力偏置机制」。
- **T5**：更彻底的简化。只在 attention 矩阵上加一个依赖相对距离的**可训练标量偏置**，相当于给 attention score 加一张「距离查分表」。还做相对位置**分桶**：近距离分得细（`0~7` 各一个桶），远距离分得粗（多个距离共用一个桶）。像地图比例尺——找门牌号要细分辨率，判断「在城东还是城西」不必精确到米。
- **DeBERTa**：和 T5 几乎相反，强调「内容-位置」和「位置-内容」的交互（disentangled attention），把「内容是什么」和「处在什么相对位置」拆开再交互。它给了一个有启发的视角：大多数任务靠相对位置，但某些场景绝对位置也有价值，于是前面层用相对、后面少数层补绝对。「绝对还是相对」从二选一，变成「不同层可能需要不同位置知识」。

把这几条放一起，趋势很清楚：相对位置编码越来越少把位置当静态标签去记，越来越多把它当成「注意力里两两关系的修正项」，角色从「输入属性」转向「打分规则的一部分」。这也是为什么后面 RoPE、ALiBi 显得自然:它们延续了这条思路，只是把「关系」写进了不同位置。

## RoPE 与 ALiBi：两种收束

RoPE 和 ALiBi 是这条演进线上的两个终点，路线很不同。

**RoPE**（MiniMind 用的）一句话：**通过绝对位置编码的方式实现相对位置编码**。它不另造相对位置表，而是把位置 `m`、`n` 分别作用到 `q_m`、`k_n` 上——把一对通道看成复平面上的向量、乘一个与位置相关的旋转因子（机制见主线 [03-rope](../02-model/03-rope.md)）。由于 attention 看的是旋转后 Q/K 的内积，这个内积天然只和相对角度差 `(m−n)θ` 有关，于是「用绝对位置操作，得到相对位置效果」。它兼顾「知道自己在哪」和「知道相差多远」，且是正交变换、不改向量模长、不易破坏原模型稳定性——这几点让它成了今天大量开源 LLM 的标配。

**ALiBi** 直接得多：不碰 Q/K，而在 softmax 前把分数减去一个**与相对距离成正比的线性惩罚**。每个 head 斜率不同，有的偏好近处、有的看得远一点。它像局部注意力的「平滑版」——不硬裁远处，而是距离越远扣分越多。因为惩罚直接加在 score 上，实现非常干净，不改 embedding、也不必为更长长度重学参数，天然带一点外推味道。

两者其实在回答同一个问题:attention 不懂顺序，位置关系写到哪里。**RoPE 写进 Q/K 的几何结构，ALiBi 写进打分规则**。差别不只在数学形式，更在设计哲学：前者相信几何表达力（还能继续做 [长度外推](03-rope-length-extrapolation.md) 的各种缩放修补），后者相信距离先验足够简单直接。

## 落回 MiniMind

MiniMind v2/v3 都用 **RoPE**。放进这条谱系看，它的选择正是主流那一支：

- 不选绝对位置（正弦/可学习）——它们把位置加在 embedding 上、会被后续层稀释，长度外推还差；
- 不选 T5 式相对偏置——那要给 attention 加一张可训练距离表，多一套参数；
- 选 RoPE——位置以旋转注入 Q/K，点积里自然浮现相对位置差，不加参数、不改 attention 主体，还能顺着接 YaRN 那套长度外推。

于是 MiniMind 在位置编码史上的坐标很清楚：**它站在「相对位置写进 attention」这条线的终点 RoPE 上**。往前看是绝对→相对的整段演进（本篇），往后看是 RoPE 之上的长度外推（附录 03），中间是 RoPE 本身的旋转机制（主线 03）。

## 一张表看懂演进

| 方案 | 位置写在哪 | 相对前一步的关键改动 | 长度外推 | 代表 |
|---|---|---|---|---|
| 正弦绝对 | 加到 embedding | —（基线：显式函数生成坐标） | 理论可，但结构易被稀释 | 原始 Transformer |
| 可学习绝对 | 加到 embedding | 改成**为每个位置学参数** | 差（超训练长度无参数） | BERT |
| RPR / Transformer-XL | attention 分数 | 位置**移进 attention**、只改打分不改 value | 中 | Transformer-XL |
| T5 相对偏置 | attention 分数 | 简化成一张**可训练距离查分表** + 分桶 | 中 | T5 |
| RoPE | 旋转 Q/K | 用**绝对旋转**换来相对效果、不加参数 | 好（可接 YaRN） | **MiniMind**、LLaMA/Qwen |
| ALiBi | softmax 前的分数 | 换成**线性距离惩罚**、极简 | 好（天然） | BLOOM 等 |

读这张表：每一行都在回答上一行的短板——绝对位置会被稀释、外推差，就把位置移进 attention；相对位置表参数多，就简化成距离偏置；还想不加参数又兼顾绝对与相对，就用 RoPE 的旋转。MiniMind 停在 RoPE 这一格。

## 常见误区

- **「RoPE 是绝对位置编码」**——它用绝对位置**操作**（按 m、n 旋转），但进入 attention 分数后体现的是相对位置差，所以本质是「用绝对手段实现相对效果」。
- **「相对位置编码要改 value」**——现代做法基本不改。Transformer-XL 起就明确：位置只加到 attention 分数、决定「看谁」，不混进被加权汇总的 value。
- **「正弦位置编码天然会外推」**——理论上有结构、可外推，但实际把位置向量混进 embedding、过投影后相对结构会被冲淡，外推并不稳。
- **「ALiBi 比 RoPE 落后」**——不是落后，是另一条设计哲学（距离先验 vs 几何旋转）。只是 RoPE 的几何表达 + 可缩放外推让它成了更主流的选择。

## 练习

1. 为什么 self-attention 必须额外注入位置信息？「permutation-equivariant」在这里指什么？
2. 位置关系的三类（绝对/相对/距离）各是什么？为什么说好的位置方案要兼顾三者？
3. 绝对位置编码的两条路线（正弦 / 可学习）各自的优劣是什么？为什么可学习式长度外推差?
4. 相对位置编码从 RPR 到 T5，主要在简化什么?为什么说「位置越来越像注意力偏置」?
5. RoPE 为什么被说成「用绝对位置实现相对位置」?它相比 T5 式相对偏置省了什么?MiniMind 为什么选它?

<summary>参考答案</summary>

1. 因为 attention 的 `QK^T` 只看内容匹配、不含顺序，打乱 token 顺序结果不变（permutation-equivariant：输入置换则输出跟着置换，但模型意识不到原有语序）。语言的顺序承载语义，所以必须额外注入位置。
2. 绝对=在第几格（坐标）；相对=在我前面还是后面几位（关系）；距离=相差多远（依赖强度）。不同语言规律侧重不同（文档首尾靠绝对、修饰关系靠相对、远程依赖靠距离），好方案要同时兼顾，还要能长度外推。
3. 正弦式由显式函数生成、有结构先验、理论可外推，但混进 embedding 后相对结构易被稀释、不区分前后；可学习式为每个位置学参数、更贴训练任务，但只「认识」训练见过的位置，超出长度没参数，外推差。
4. 从「给每个相对距离配一个向量」（RPR）简化到「只加一个可训练标量距离偏置 + 分桶」（T5），参数越来越少、越来越集中在 attention 打分侧。Transformer-XL 起明确位置只改 attention 分数、不改 value，所以位置越来越像一种「注意力偏置机制」。
5. 因为它按各自绝对位置 m、n 旋转 Q、K，但点积后只剩相对角度差 (m−n)θ，用绝对手段得到相对效果。相比 T5 它不需要一张可训练距离表、不加参数，也不改 attention 主体。MiniMind 选它因为：不被后续层稀释、不加参数、点积自然浮现相对位置、还能接 YaRN 做长度外推。
